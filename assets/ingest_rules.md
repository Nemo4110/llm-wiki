# Ingest 规则

> 指导 Agent 如何处理不同类型的资料。

## 通用原则

1. **提取而非复制**：不要大段复制原文，要提取结构化洞察
2. **连接而非孤立**：每个新概念都要链接到现有概念
3. **溯源**：每个声明都要能追溯到具体来源

## 按资料类型的处理策略

### 学术论文

**提取内容**：
- 研究问题
- 方法创新
- 关键结果
- 局限性和未来方向

**创建页面**：
- 论文同名页面（包含完整信息）
- 方法页面（如果方法是通用贡献）
- 更新相关领域综述页面

### 技术博客/文章

**提取内容**：
- 核心观点
- 代码模式（如果有）
- 实践经验/教训

**创建页面**：
- 技术主题页面
- 模式库（如果是实现技巧）

### 代码仓库

**提取内容**：
- 架构设计
- 关键抽象
- 依赖关系

**创建页面**：
- 项目页面
- 设计模式页面（提取通用模式）

### 笔记/想法

**提取内容**：
- 原始洞察
- 待验证的假设
- 问题/待办

**创建页面**：
- 草稿页面（标记为 draft 状态）
- 或追加到相关现有页面

## 页面创建决策树

```
新概念？
  ├─ 是 → 是否已存在相关页面？
  │        ├─ 是 → 更新现有页面，添加链接
  │        └─ 否 → 创建新页面
  └─ 否 → 更新现有页面
```

## 合并策略

当新资料与现有页面相关时：

1. **补充**：添加新信息到合适段落
2. **对比**：如果新资料有不同观点，添加「不同视角」段落
3. **更新**：如果新资料是更新的版本，替换旧信息并注明
4. **冲突标记**：如果存在矛盾，保留两者并标记 `⚠️ 矛盾：来源X说...，来源Y说...`
